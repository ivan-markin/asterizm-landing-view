Why Asterizm?

Privacy

With Asterizm, you can transfer sensitive data to other networks without the risk of compromise at the Relayer level.
No more front-running and data leaks!

High Speed
Low Cost

Thanks to the encryption-based protocol architecture, we don’t maintain a complex economic system to achieve consensus. This allows all cross-chain transactions to flow almost instantly and at one of the lowest fees on the market.

Flexibility

The architecture allows to embed Asterizm Client module into any infrastructure used by the client dApp in compliance with the security policies and access levels.

Scalability

Asterizm is easily scalable — an increase in the number of transactions does not affect the stability of the protocol, the speed or the cost of transactions.

Step 1

The client implements a smart contract using an abstract class from Asterizm and deploys it to the required networks. The _initAsterizmTransferEvent() method is used to initiate a cross-chain transaction on the contract. The data and parameters of the cross-chain transaction are transmitted to it. The transaction can contain arbitrary information and one or multiple actions (instructions) to be performed on the destination networks.

Step 2

The client’s smart contract sends the obtained data to the client’s off-chain module, generating a unique xID based on the transaction content, source address, destination address, source network ID, destination network ID and a unique transaction number.

Step 2a

If there is a need to send assets together with a cross-chain message, the client off-chain module initiates a cross-chain swap rate request via API. On receipt of calldata, the module sends it to the client smart contract, and after that, the tokens are sent to the Asterizm module's cross-chain swap contract.

Step 3a

The client's assets arrive at the target contract in the destination network.

Step 3

The client’s off-chain module encrypts the cross-chain transaction content and sends it back to the client’s smart contract in the source Initializer network, calling the cross-chain transfer initialization method.

Step 4

The client’s smart contract sends the encrypted data with transaction parameters to the Initializer smart contract for verification.

Step 5

The Initializer smart contract checks the transaction nonce to preserve the cross-chain transaction execution sequence and transmits the encrypted data with the unique parameters to the Transmitter smart contract.

Step 6

Asterizm Relayer servers pull the encrypted data with the parameters from the Transmitter contract and send it to the Transmitter smart contract in the destination network for further processing.

Step 7

The Translator smart contract on the destination network accepts the encrypted data with the parameters from the Relayers and passes it to the Initializer smart contract for validation.

Step 8

After receiving the encrypted data and the parameters, the Initializer smart contract checks the nonce to comply with the transaction sequence, stores the transaction xID to validate the transaction at step 10, and transmits the data to the client’s smart contract.

Step 9

The client’s smart contract in the destination network receives the encrypted data with the parameters, creates an event, and transmits the data to the client’s off-chain module for verification and decryption. Note: This is where the first step of cross-chain transaction validation is performed by matching the xID generated in the source network at the transaction initialization, and by checking the source network and address. The client’s server verifies that the data is complete and consistent, thus eliminating the risk of compromise at the Relayer server level.

Step 10

After data verification and decryption, the client’s server initiates a transaction on the client smart contract in the destination network, sending the decrypted data and xID to the contract, and validating the transaction on the Initializer smart contract by checking the hash from the decrypted data and the xID, which was previously sent together with the transaction through the Initializer contract. Note: At this step, the integrity of the data and the trusted addresses is checked before the transaction is executed. This check eliminates the risk of hacking the client’s server. Deep tech: The verification is performed by checking the hash from the decrypted data and the xID that was previously stored on the Initializer contract. If the verification succeeds, it means that the client’s contract executes exactly the transaction it sent, and does so for the first time. This algorithm eliminates the possibility of spamming transactions and spoofing data on the client’s server (if, for example, the server is compromised).

Step 11

The client’s smart contract sends the encrypted data with transaction parameters to the Initializer smart contract for verification.

Go cross-chain in 3 easy steps

Set up the off-chain module

Implement Client module (Docker image) on your server to encrypt and decrypt messages with SHA256.

Deploy smart contract in preferred chains

Build smart contracts with cross-chain logic extending the Asterizm client interface. The encryption algorithm can be changed according to the company's security policies or requirements.

Run cross-chain operations!

Send and receive your first cross-chain message by calling the _initAsterizmTransferEvent() method on your Asterizm Client Contract
